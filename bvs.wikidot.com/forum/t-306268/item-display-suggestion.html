Item display overhaul
taltamir 08 Feb 2011 05:41

Originally this was a suggestion (really more of a request for someone to do the work). But I spent the last 5 and a half hours learning and then doing it (instead of my homework T.T) So here is my work for all to see.
I went ahead and created overhauled include pages (I did not modify the current include:item, but instead created include:item-black, include:item-red, include:item-blue)
Those show the exact details and near exact appearance of hovering over an item, and of left clicking an item.

I have applied each new template to one (matching) item as a "demo":
For a black item (non perm, non consumable):
billy-bucket
For perm item:
regalia
For consumable item:
super-potion

I think it looks AMAZING and definitely worth it to switch all pages to be this way. The neatest thing is that its only done via simple code and no images so it should be super light.
If someone knows how to make the table appear on the top right (Where the image used to be), please modify the include pages to do so.

Note that these are exactly as they are in the game, the red items show their mouseover data in their leftclick, the blue do not, the black do not even have mouseoever.
The quickest way to update all the pages with the needed info would be to change the item page to say:
This page is using an obsolete item include, please replace with the current include from Item Template

However, I think I think for such a far reaching change (it will retroactively affect all item pages) a moderator's permission should be sought.

Here is (most of) the info I collected to do it:

Perm's mouseover:
Pink background: F8D5E7
White background: FEFCFA
Outline: E9A6CC
Text on white: 6C244D
Text on Pink: 9C527B
Width (is always): 306 pixels

Non-perm mouseover:
same as perm, except colors are different, I inputted the data directly into the include:item-blue page

Item left-click description:
Blue Background: 0E0C5F
Brown background: E0C69A
Text on brown: 000000
Tradeable background: 660000
Loopable background: 006600
Permanent background: 666600
Sellable background: 000066
White text: FFFFFF
Dotted border: A75928
Image Wideth= 100px +3 on each side
width of rest= 192px
pixels after the 106 of picture to reach tradeable/sellable box: 17px
pixels between tradeable and loopable: 2px
Tradeable text: Tradable / Untradable
Sellabele text: Is Sellable / Not Sellable
Loopable text: Is Loopable / Not Loopable
Permanent text: Permanent / Non-Permanent
Image border: black aka 000000

The current models are available at:
item
item-red
item-blue

problems:
Issue 1: putting 306 pixels in the table style gives me a 300px wide cells. Putting 306px in the cell style gives me 314px wide cells. Rendered moot by issue 2.
Issue 2: The text on the wiki is different (larger, wider spaced) then the one used for the mouseover & leftclick boxes. So if someone wants to tweak them to match go right ahead. As it is, the boxes are similar but not exact to what is found in BvS.

I realized, the pink popup is only for items colored red in your bucket. Blue colored ones have a light blue popup, and black colored ones have no popup. I created seperate colored ones.
AFAIK the reds are pemanent items, and blue are consumables, while black items are neither.

hi, thanks for everything.
The float:right function is a great addition, makes it much better. There are two more thing you can help with (sorry for wall of text, its at the bottom)

For most items the tooltip are indeed redundant as you say. But some items need it.
Take for example the 5 essences.
The left click box in BvS says:
Item Description N/A
Description coming soon!

In the left click box. The only info the left click box gives you is if it is tradable, sellable, loopable, and permanent. If it was the only thing we had, it would not clarify what the item does.
The mouseover is what actually gives you the properties of using the item; this is all too common for consumables. Perms typically have their fluff description AND mouseover description in the same box (with specific formatting of bold, italics, and parenthesis). However there are exceptions, tacos are a consumable but are formatted like a perm; there might be perms formatted like consumables with missing info only available from floatbox.

We could simply enter a description in there based on what the item is rather then what info it presents in the BvS, although that brings other issues. Or we could only include the float boxes on pages where they are needed.

Issue 1:
Speaking of the 5 essences, there is an issue with them essence-of-earth
The "consumption box" is clipping over the floated to the right leftclick box.
Issue 2:
What do you think about floating the mouseover tooltip to the right as well? I looked into it and floating the mouseover box to the right places it to the left of the current float, rather then below it.
