Exa

Exa is an ls replacement for Gnu/Linux based operating systems, written in Rust. It primarily supports the same flags as ls does, which I will describe below.

-l, the Long List

-l, which is short for –long, changes how the list is output. By default, exa will output folders and files in a multi-column like manner, based on the size of the file names involved. It will generally not show any of the files properties other than those that can be conveyed by color (different filetypes == different file formats)

The -l flag changes this behavior to show one entry per line, with a lot of extra information.

If you want to learn what all the columns are for, use the -h flag

-a, the All Files

-a, which is short for –all, includes files that would otherwise be hidden, such as dotfiles and hidden directories. There isn’t really much to this command

-h, the Headers

-h, or –header, adds an extra line at the top of the output, defining the columns directly in the terminal, just in case you forget what they are.

–git, Git Status for Plebs

–git will list each file’s git status, if it is tracked or ignored
